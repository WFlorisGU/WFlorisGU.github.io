下面记述一下，如何在vba中实现汇总word文章中不同表的内容资料：

` Sub 汇总所有数据()
‘定义所有变量
Dim ff As String, path As String, a As Integer, Filename As String
‘定义等会窗口中过滤出的文件类型
ff = “Word文档(.doc;.docx),.doc;.docx “
‘窗口中左上角显示文字
strtitle = “打开Word文档”
‘选择后取路径，并赋值到path中
path = Application.GetOpenFilename(filefilter:=ff, Title:=strtitle)

‘将“路径”中的路径和文件名整体分开
a = Len(path)
For i = 1 To a
‘从右开始直到截取到\，才执行if程序中内容
If Mid(Right(path, i), 1, 1) = “\” Then
path = Mid(path, 1, a - i + 1)
Exit For
End If
Next
‘按照dir排序path所有文件名
Filename = Dir(path)

'循环查找word文件中table中的指定值，输出到excel中
k = 2
'开始循环，文件不为空，直接开始循环
Do While Filename <> ""
 '下面这一条固定，表示word应用程序（docapp）被创建,同样每一循环都必须打开word
 Set docapp = CreateObject("Word.application")
 '创建对象，打开word文档
 Set dd = docapp.documents.Open(path & Filename)
 '创建对象，附上文档中的第一个表格table
 Set ddd1 = dd.Tables(1)
 Set ddd2 = dd.Tables(2)
 '计算表格中行数
 x = ddd2.Range.Rows.Count
 For i = 2 To x    'i为word中表格的行数
  Cells(k, 1) = Mid(ddd1.cell(1, 2).Range, 1, Len(ddd1.cell(1, 2).Range) - 2)
  Cells(k, 2) = Mid(ddd2.cell(i, 2).Range, 1, Len(ddd2.cell(i, 2).Range) - 2)
  Cells(k, 3) = Mid(ddd1.cell(1, 4).Range, 1, Len(ddd1.cell(1, 4).Range) - 2)
  Cells(k, 4) = Mid(ddd1.cell(1, 6).Range, 1, Len(ddd1.cell(1, 6).Range) - 2)
  Cells(k, 5) = Mid(ddd1.cell(2, 4).Range, 1, Len(ddd1.cell(2, 4).Range) - 2)
  Cells(k, 6) = Mid(ddd1.cell(3, 4).Range, 1, Len(ddd1.cell(3, 4).Range) - 2)
  Cells(k, 7) = Mid(ddd2.cell(i, 3).Range, 1, Len(ddd2.cell(i, 3).Range) - 2)

  k = k + 1      'k为输出到的表格行数
 Next
 '开完一个文档后，各种关闭和归零
 dd.Close
docapp.Quit
 Set docapp = Nothing
 Set dd = Nothing
 Set ddd1 = Nothing
 Set ddd2 = Nothing
 '文件到下一个
 Filename = Dir
 Loop

 '所有文件读取完后，手动修改第一行
 Cells(1, 1) = "姓名"
 Cells(1, 2) = "日期"
 Cells(1, 3) = "性别"
 Cells(1, 4) = "学历"
 Cells(1, 5) = "工资"
 Cells(1, 6) = "年龄"
 Cells(1, 7) = "销售客单"

End Sub `

127小时

发表于 2018-01-05

不想悟出点什么“不要一个人什么都不说就出去（消失）”
但是这真的会让你爱的人伤心难过。
Aran在困到意识模糊的时候最后最后看到的都是自己爱的人和自己内心小孩。爱的人提醒你，他们一直在你的身边，无论你发生什么。从你出生到现在，唯一不变的就是那个内心小孩，他一直在你的心里，他懂你。真的是懂你。127个小时真的是全然和自己独处，从一开始的愤怒，挣扎到昏迷痛苦再到后面的绝望麻木，等待死亡。一部电影就把这三个阶段全部全部表现出来。一个经历的体悟只能来自你自身，别人是无法与你共享的，即使能够传达，也没有什么百分之百。
其中Aran在最绝望的时候，在思考这一切是不是一种宿命，自己打从一开始，出生的那一刻就是等待着这颗把他压在峡谷深处的大石头，等着他长大，踏入这个峡谷，然后没踩稳与这个石头一同掉落，手被石头压住，被困……自己经历的一切难道是一种思维定势，我所有的认知习惯，行为特点冥冥之中都是在一种模式中，这一系列的东西把我带到这里，带向这一切的结果。这真的是“我们都生活在我们想象的世界里面……”你的思考，你的动机，你的一切行为看似没有规律，其实都在那里，等着你。
自己很感动同时也不敢看下去的那一段是最后Aran自己截肢的哪一个片段。他最后看到他的内心小孩，朝他笑，他自己也在想象自己和那个小孩在一起玩耍，但是最后内心小孩却跑走了。Aran不甘心他想要去看，他想要再和自己的小孩在一起，他努力，他下了决心，做了自己之前不敢做的事情“截肢”……对于现在的我很难敲下什么东西去描述那个过程。我真的真的不敢也不想去想象背后的东西。我只想在抱抱他，那个内心的小孩。
很多时候，我们都说我们的一生想要获得快乐，幸福。很多人穷尽一生都在追求这个，甚至到了很极端的道路。电影里面有很多让我很很感动的地方。就像是被困的时候，每天能够享受的15分钟阳光，就是Aran很开心的事情。看着阳光慢慢照进峡谷，一点一点的移动，慢慢地来到你的面前，那一刻你想到的是感谢，开心和享受。能看到的乌鸦都能成为Aran在摄像机前说的话题，独处的时候，往往能注意到很多东西……其实我感觉这电影告诉我们，我们追求的其实不是幸福自己想要的刺激兴奋，而是真实，生活的真实，爱的真实。这些就够了吧。
如果可以，我不希望成为Aran，我要自己好好的活着，爱我所爱，做我所想。感恩生命，感谢自己每一天。

####协议和服务
协议：是通过双方共同遵守的规则和规定。不同节点对等层之间的通信。
Ps:结点和节点的区别：结点是节点的不正式说法。在网络中除了链路就是节点，如各种设备，路由器，交换器，计算机……
1.语法
2.语义
3.时序
服务：同一节点相邻节点的通信。相邻层形成了一种请求服务（服务数据单元SDU（Service data Unit））和提供服务（服务访问点SAP（Service Access Point））的组合。
通信有面向连接和无连接之分。就是是否需要连接的建立和释放。

####OSI参考模型：

物理层：在物理传输介质上可靠，透明的进行比特流收发。（物理连接）

++功能++
利用机械，电气，功能，规程特性在DTE和DCE之间实现对物理信道的简历，保持和拆除功能。
++基本特性++
机械特性：说明接口所用连接器的形状和尺寸，引脚数目和排列固定和锁定装置等。（可以想象电脑后面的插孔）
电气特性：说明在接口电缆的哪条线上出现的电压的范围。
功能特性：说明某线上面出现的电平的电压是什么意义
规程特性：说明不同功能各种可能发生事件出现的顺序
++DTE和DCE++
DTE（DATA TERMINAL EQUIPMENT）:数据终端设备，数据的接收或发送端，这种设备具有处理数据和发送数据的功能。通过DCE连接到网络。针头。
DCE(DATA COMMUNICATION EQUIPMENT)：数据通信设备，连接DTE的设备。提供信号变换和编码功能，负责建立，保持和释放电路的连接。提供时钟同步（所以DTE不需要有“时钟速率”）。孔头。

数据链路层：在相邻结点治安可靠的传输数据帧。

++功能++
1.链路管理：链路的建立，维持和释放
2.帧同步：指同步传输，接收方可以从收到的比特流中准确的区分出一帧的开始和结束    ps：同步：接收方按照发送方发送的每个位的起止时刻和速率来接收数据，不得有误差。
3.流量控制:控制发送速率；
4.差错检验：采用一定的编码技术，使得被检验出错的帧及时重发。
5.数据和控制信息的区分：方便接收方区分分开数据和控制信息。
6.透明传输：透明指事物没有产生任何影响，即可视为”透明“。指的是无论数据如何都可在通道内传输，同时保持为原始数据。   ps：若某比特组合和控制信息一样，则可采取零比特填充或是字符填充的方法。
7.寻址：保证送到正确的目的地。
++通信控制规程（数据链路层协议）++
分为两类：
1.面向字符型的规程
链路上传输的数据必须是由规定的字符集组成。
2.面向比特型的规程
信息传输统一以帧为单位。特点是：报文的数据和控制信息完全独立，具有良好的透明性。
++注：++
比特和字节和字符的区别和联系：
比特是信息传输的单位，0和1，单位bite。
字节是计算机中储存单位，单位byte。
字符由两个字节组成。
8比特=1字节
16比特=2字节=1字符
++BSC协议++
BSC协议（Binary Synchronous Communication protocal）字符型的规程。一种应答系统。
数据链路两端是DCE和DTE。

++流量控制++
流量控制是采取一定的措施来限制发送方的所需要的数据流量，使其发送的速率不要超过接收方能处理的速率。经常采用的处理方法就是：停止-等待协议和滑动窗口控制。
1.停止和等待协议协议。
基本原理：发送方每发送完一帧后，必须等待接收方收妥应答到来后，才能够继续发送下一帧数据。适用于单工或是半双工。（一定要受到对方确定信息）控制程序设置两个变量，发送端设置发送序号变量V（S），用来区分前后发送的两帧；接受端设置接受序号变量V（R），表明当前要接收的帧序号，以防重复接收。ACK：acknoledgement。
2.滑动窗口协议
基本思想：发送方可以连续发送若干帧不等待对方应答。显然连续发送不等待应答帧的数量必须加以限制，否则一旦传输出现错误，必须全部重发。（key：任何时刻发送的进程必须保持一组序号，对应于允许发送的帧。这些帧称为在发送窗口之内。）
Eg：已发送0-7帧，但确认收到0，2，3.问应该重发几个？ （4个）
在停止等待协议的基础上，在等待第一个确认的时间内，不断发送之后的数据。若错帧则重传。应该以收到的最大的序号为确定。

HDLC的基本工作原理
HDLC：high-level data link control
（插个18.1图）
在屋里链路上承载信息过程中的作用分为主站，从站，复合站。
主站：控制目的站的站点，主要功能是发送命令，接收响应，负责对链路的全面管理。
从站：在链路上接收数据的站点。
复合站：同时具有主站和从站的功能。
非平衡结构：具有一个主站或是多个从站连接到一个线路中
平衡结构：该结构有两个复合站的点对点连接构成。
HDLC的帧结构
标志字段：用特殊序列01111110标志帧的开始和结束。为了保证除了标志序列之外的部分不出现这个特殊序列。采用0比特插入记录：出现了5个1就在后面插入一个零。在接收端接收到信息后，利用0比特删除技术，将0删掉。
地址字段:在使用非平衡配置是，地址字段总是要写入从站的地址；在平衡配置中，总是写入应答站的地址。全1是广播地址，全0是无效地址。
控制字段：用于表示所使用帧的类型和序列号。（帧的类型：信息帧，监督帧，无编号帧。）
信息字段：实际要传输的内容。
帧校验序列：采用CRC校验方法，生成多项式。

网络层：实现通信子网的控制，控制路由选择，差错控制，拥塞控制等，把分组从源点传输到目的地点

主要功能就是：实现整个网络系统内连接，实现数据传输的透明性。
1.为运输层提供服务
2.路径选择。通向目的节点的最佳路径。
3.流量控制
4.网络连接的建立，保持和释放问题。

数据交换技术
“交换”就是按某种方式动态分配传输线路资源。
报文交换，在源端和目的端之间进行通信。在报文头上填上目的端的地址，在经由路径选择算法将报文传过去。
分组交换，在一个主机向另一主机传输数据时，将数据划分为一个一个分组。每个分组都有地址信息，根据目的信息将分组发向目的节点。这些分组储存在节点的内存中。
交换机。

数据交换方式比较
1.电路交换和报文交换
2.虚电路和数据报（分组交换网络提供的两个服务）
数据报，主机想发数据随时就可以发，分组发送自己选择路径，所以在目的机的分组顺序不能保证。因此节点还需要缓存排好顺序。但当网络拥塞时，分组会被丢弃。所以这是不可靠传输。
虚电路，源主机发送虚呼叫，和目的主机之间建立最合适的虚电路。之后分组都在此虚电路上传输。传输结束后，释放电路。（由于采用的存储转发的分组交换，所以只是断断续续的占用一条线路）
++路由算法++
分组交换网的路由选择是通过查找路由表来完成的。在路由表中仅登记发往的目的交换机号就好了。

拥塞控制

运输层：为高层用户提供可靠的端对端的通信服务。

会话层：允许不同主机上各种进程之间的会话通信。

表示层：为上层用户提供格式转换，编码，压缩，加密等服务。

应用层：定义应用的框架。

第十届网络商务创业大赛后记

发表于 2017-11-04

现在的我在自己的小空间里面，敲下自己在这个项目中的成长和反思。一边敲感觉橘黄色的灯光仿佛也有阳光的温度了，感觉超级好~

“我做了什么 我为什么做 我学到了什么 我学到的这些可以教会我什么”

于我而言，这个项目我感觉是我在大学里面第一次正式的接触项目，开始撰写方案。第一次研究速卖通，第一次接触学校里面创业的人。创造了很多的第一次吧。在比赛方案中自己负责了方案中市场调研，盈利模式分析，营销模块，平台实践方面的调查和截图等等。
我之所以参加比赛，也是因为受到了各种奖学金自己没有填好，比赛方面真的很欠缺的问题刺激，就……报了比赛。可以说是我的心机不纯吧，但是结果真的也是还行了：省二等奖。算是我的目的达成了，同时一路上学到了很多东西啊。太值了。

学到的东西主要分两个方面来写吧。
项目知识方面
1.一个方案的撰写。很多部分基本的框架，部分内容怎么写，各部分内容细节，如表和图号码，来标识源。对文字的可视化处理。看待问题角度的重要性！
2.一些小模块的知识点。项目的可行性（四个角度）、项目的盈利分析、市场分析、对很重要的SWOT分析（四个角度：优，劣，机会，威胁，两个方面：内部，外界）。各种营销方案：重点整合营销，将原本独立的几个营销手段有机结合起来，实现1+1>2的结果。真正的去做出来。
3.跨境电商行业很浅的知识。了解了很多跨境电商的行业背景，目前存在的平台。速卖通的各种“打法”（才知道淘宝后面的一些“黑幕”）。商品标题的设置技巧等各种有关访问量的关键点。扩展了很多见识，换了一个角度去面对同一件事情“网上购物”。自己知道的只是冰山一角啊。
团队方面++
知道自己作为一个队员应该是什么样的角色。在本次项目中，作为我们队伍的一名队员，自己更应该去相信团队，相信每个人担当的责任和角色。团队之间的相信很重要。作为一个爱操心的人，一定要记得“严于律己，宽以待人”！！！做好自己的事情，实时适当的言语指出问题，最后还是要回归到做好自己的事情。团队的力量是重要的。（突然意识到团建也好重要啊）放心放心，下次做的更好。

学到的这些当然是锻炼了我的能力啊，学习能力，沟通能力，应用能力，合作能力……
在这次里面很棒的是把自己在课上的知识用了出来，真正的实践出来。在这次的方案撰写中，发现了自己在做文字可视化的处理，编辑方面做得不错。
当然，在这次的项目中，发现了自己很多需要学习的东西。如如何用专业化的语言撰写方案。分析问题的角度（这一方面可能是自己了解时间项目还不够深吧）。各个模块的知识自己都还学得太浅，很多东西都是博大精深。都需要去了解去看，如营销方面的各种营销手段，网红营销，社交工具营销等。分析项目的工具等等。
很重要的是接触到了跨境这个新的行业，速卖通这个平台，也能明白为什么之前家里用淘宝买茶叶效果不高。（T0T）

总结而言就是自己要学的东西真的太多的~
世界那么多新玩意~太棒了！
