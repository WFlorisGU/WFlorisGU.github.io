2017-11-06 PV:

####协议和服务
协议：是通过双方共同遵守的规则和规定。不同节点对等层之间的通信。
Ps:结点和节点的区别：结点是节点的不正式说法。在网络中除了链路就是节点，如各种设备，路由器，交换器，计算机……
1.语法
2.语义
3.时序
服务：同一节点相邻节点的通信。相邻层形成了一种请求服务（服务数据单元SDU（Service data Unit））和提供服务（服务访问点SAP（Service Access Point））的组合。
通信有面向连接和无连接之分。就是是否需要连接的建立和释放。

####OSI参考模型：

物理层：在物理传输介质上可靠，透明的进行比特流收发。（物理连接）

++功能++
利用机械，电气，功能，规程特性在DTE和DCE之间实现对物理信道的简历，保持和拆除功能。
++基本特性++
机械特性：说明接口所用连接器的形状和尺寸，引脚数目和排列固定和锁定装置等。（可以想象电脑后面的插孔）
电气特性：说明在接口电缆的哪条线上出现的电压的范围。
功能特性：说明某线上面出现的电平的电压是什么意义
规程特性：说明不同功能各种可能发生事件出现的顺序
++DTE和DCE++
DTE（DATA TERMINAL EQUIPMENT）:数据终端设备，数据的接收或发送端，这种设备具有处理数据和发送数据的功能。通过DCE连接到网络。针头。
DCE(DATA COMMUNICATION EQUIPMENT)：数据通信设备，连接DTE的设备。提供信号变换和编码功能，负责建立，保持和释放电路的连接。提供时钟同步（所以DTE不需要有“时钟速率”）。孔头。

数据链路层：在相邻结点治安可靠的传输数据帧。

++功能++
1.链路管理：链路的建立，维持和释放
2.帧同步：指同步传输，接收方可以从收到的比特流中准确的区分出一帧的开始和结束    ps：同步：接收方按照发送方发送的每个位的起止时刻和速率来接收数据，不得有误差。
3.流量控制:控制发送速率；
4.差错检验：采用一定的编码技术，使得被检验出错的帧及时重发。
5.数据和控制信息的区分：方便接收方区分分开数据和控制信息。
6.透明传输：透明指事物没有产生任何影响，即可视为”透明“。指的是无论数据如何都可在通道内传输，同时保持为原始数据。   ps：若某比特组合和控制信息一样，则可采取零比特填充或是字符填充的方法。
7.寻址：保证送到正确的目的地。
++通信控制规程（数据链路层协议）++
分为两类：
1.面向字符型的规程
链路上传输的数据必须是由规定的字符集组成。
2.面向比特型的规程
信息传输统一以帧为单位。特点是：报文的数据和控制信息完全独立，具有良好的透明性。
++注：++
比特和字节和字符的区别和联系：
比特是信息传输的单位，0和1，单位bite。
字节是计算机中储存单位，单位byte。
字符由两个字节组成。
8比特=1字节
16比特=2字节=1字符
++BSC协议++
BSC协议（Binary Synchronous Communication protocal）字符型的规程。一种应答系统。
数据链路两端是DCE和DTE。

++流量控制++
流量控制是采取一定的措施来限制发送方的所需要的数据流量，使其发送的速率不要超过接收方能处理的速率。经常采用的处理方法就是：停止-等待协议和滑动窗口控制。
1.停止和等待协议协议。
基本原理：发送方每发送完一帧后，必须等待接收方收妥应答到来后，才能够继续发送下一帧数据。适用于单工或是半双工。（一定要受到对方确定信息）控制程序设置两个变量，发送端设置发送序号变量V（S），用来区分前后发送的两帧；接受端设置接受序号变量V（R），表明当前要接收的帧序号，以防重复接收。ACK：acknoledgement。
2.滑动窗口协议
基本思想：发送方可以连续发送若干帧不等待对方应答。显然连续发送不等待应答帧的数量必须加以限制，否则一旦传输出现错误，必须全部重发。（key：任何时刻发送的进程必须保持一组序号，对应于允许发送的帧。这些帧称为在发送窗口之内。）
Eg：已发送0-7帧，但确认收到0，2，3.问应该重发几个？ （4个）
在停止等待协议的基础上，在等待第一个确认的时间内，不断发送之后的数据。若错帧则重传。应该以收到的最大的序号为确定。

HDLC的基本工作原理
HDLC：high-level data link control
（插个18.1图）
在屋里链路上承载信息过程中的作用分为主站，从站，复合站。
主站：控制目的站的站点，主要功能是发送命令，接收响应，负责对链路的全面管理。
从站：在链路上接收数据的站点。
复合站：同时具有主站和从站的功能。
非平衡结构：具有一个主站或是多个从站连接到一个线路中
平衡结构：该结构有两个复合站的点对点连接构成。
HDLC的帧结构
标志字段：用特殊序列01111110标志帧的开始和结束。为了保证除了标志序列之外的部分不出现这个特殊序列。采用0比特插入记录：出现了5个1就在后面插入一个零。在接收端接收到信息后，利用0比特删除技术，将0删掉。
地址字段:在使用非平衡配置是，地址字段总是要写入从站的地址；在平衡配置中，总是写入应答站的地址。全1是广播地址，全0是无效地址。
控制字段：用于表示所使用帧的类型和序列号。（帧的类型：信息帧，监督帧，无编号帧。）
信息字段：实际要传输的内容。
帧校验序列：采用CRC校验方法，生成多项式。

网络层：实现通信子网的控制，控制路由选择，差错控制，拥塞控制等，把分组从源点传输到目的地点

主要功能就是：实现整个网络系统内连接，实现数据传输的透明性。
1.为运输层提供服务
2.路径选择。通向目的节点的最佳路径。
3.流量控制
4.网络连接的建立，保持和释放问题。

数据交换技术
“交换”就是按某种方式动态分配传输线路资源。
报文交换，在源端和目的端之间进行通信。在报文头上填上目的端的地址，在经由路径选择算法将报文传过去。
分组交换，在一个主机向另一主机传输数据时，将数据划分为一个一个分组。每个分组都有地址信息，根据目的信息将分组发向目的节点。这些分组储存在节点的内存中。
交换机。

数据交换方式比较
1.电路交换和报文交换
2.虚电路和数据报（分组交换网络提供的两个服务）
数据报，主机想发数据随时就可以发，分组发送自己选择路径，所以在目的机的分组顺序不能保证。因此节点还需要缓存排好顺序。但当网络拥塞时，分组会被丢弃。所以这是不可靠传输。
虚电路，源主机发送虚呼叫，和目的主机之间建立最合适的虚电路。之后分组都在此虚电路上传输。传输结束后，释放电路。（由于采用的存储转发的分组交换，所以只是断断续续的占用一条线路）
++路由算法++
分组交换网的路由选择是通过查找路由表来完成的。在路由表中仅登记发往的目的交换机号就好了。

会话层：允许不同主机上各种进程之间的会话通信。

表示层：为上层用户提供格式转换，编码，压缩，加密等服务。
